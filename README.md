# MatchApp バックエンド

位置情報ベースのリアルタイム・マッチングアプリケーション「MatchApp」のバックエンド API サーバーです。
SMS 認証によるユーザー登録・ログイン、近隣ユーザーの検索とマップ表示、マッチングリクエストと
待ち合わせ（ミーティング）管理、Socket.io によるリアルタイム通信を提供します。



---

## 主な機能

- 📱 **SMS 認証** — Twilio を利用した電話番号認証（開発環境ではコンソールにコードを出力）
- 🔐 **JWT 認証** — アクセストークン（7日）とリフレッシュトークン（30日）
- 📍 **位置情報検索** — MongoDB の `2dsphere` インデックスによる近隣ユーザー検索
- 🗺️ **マップ表示** — マーカー・距離計算（ハバーサイン公式）・設定 API
- 💞 **マッチング** — リクエスト送信／承認・拒否、待ち合わせ地点の中間点自動算出
- 🤝 **ミーティング確認** — 双方の確認による会合成立と実会数のカウント
- ⚡ **リアルタイム通信** — Socket.io によるオンライン状態・位置共有・通知
- 🛡️ **セキュリティ** — Helmet、CORS、レートリミット、エラーハンドリング

## セットアップ

```bash
# 1. 依存パッケージのインストール
npm install

# 2. 環境変数ファイルの作成
cp .env.example .env
#   .env を編集して各値を設定してください

# 3. （任意）テストデータの投入
npm run seed:map      # 東京エリアのテストユーザーを投入

# 4. 開発サーバーの起動
npm run dev
```

起動後、`http://localhost:5000/api/ヘルス` で稼働状況を確認できます。

## 環境変数（`.env`）

| 変数名 | 説明 | 例 |
| --- | --- | --- |
| `MONGODB_URI` | MongoDB 接続文字列 | `mongodb://localhost:27017/matching-app` |
| `JWT_SECRET` | JWT 署名用シークレット | （十分に長いランダム文字列） |
| `TWILIO_ACCOUNT_SID` | Twilio アカウント SID | 本番環境で必須 |
| `TWILIO_AUTH_TOKEN` | Twilio 認証トークン | 本番環境で必須 |
| `TWILIO_PHONE_NUMBER` | SMS 送信元電話番号 | `+1234567890` |
| `PORT` | サーバーポート | `5000` |
| `NODE_ENV` | 実行環境 | `development` / `production` |
| `CLIENT_URL` | フロントエンドの URL（CORS 用） | `http://localhost:3000` |

> 開発環境（`NODE_ENV=development`）では SMS は実際には送信されず、認証コードがサーバーの
> コンソールに出力されます。

## Socket.io イベント

接続時に `auth: { token }` で JWT を渡して認証します。

**サーバー → クライアント（受信）**

| イベント名 | 説明 |
| --- | --- |
| `ユーザーオンライン` / `ユーザーオフライン` | 他ユーザーの接続・切断 |
| `ユーザー位置更新` | 他ユーザーの位置更新 |
| `新規マッチリクエスト` | マッチングリクエストを受信 |
| `マッチ承認` / `マッチ確定` / `マッチ拒否` | マッチングの結果通知 |
| `ミーティング確認` | 相手がミーティングを確認 |
| `位置共有要求` / `位置共有完了` / `位置共有期限切れ` | 位置共有関連 |

**クライアント → サーバー（送信）**

| イベント名 | 説明 |
| --- | --- |
| `位置更新` | 自分の位置を更新 |
| `ルーム参加` / `ルーム退出` | ルームの参加・退出 |
| `メッセージ送信` | ルームへメッセージ送信 |
| `ミーティング接近` | ミーティング地点への接近通知 |
| `位置共有リクエスト` / `位置共有` | 位置共有の要求・実行 |

## データモデル

### ユーザー（User）

`名前, 電話番号, 性別, 住所, 位置(GeoJSON Point), プロフィール写真, 自己紹介,
オンライン状態, マッチ数, 実会数, SMS認証済み, SMSコード, SMSコード有効期限,
最終接続, ソケットID`

### マッチ（Match）

`リクエスト者ID, 対象ユーザーID, ステータス(pending/accepted/rejected/expired),
会う理由, 待ち合わせ地点(GeoJSON Point), 有効期限`

### ミーティング（Meeting）

`マッチID, 予定時刻, 実会時刻, リクエスト者確認済み, 対象者確認済み, 両者確認済み,
リクエスト者評価, 対象者評価, 会合成功, メモ`

```
├── config/
│   └── database.js        # MongoDB 接続

├── routes/                # ルート定義（auth / users / matching / map）

```
